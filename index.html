NexDB
Home Docs API Architecture

Embeddable, lightning-fast document database
with Redis caching & SQLite persistence

NexDB combines the developer experience of MongoDB (documents, collections, dynamic queries), the speed of Redis (in-memory WAL + optional caching), and the reliability of SQLite (durable SQLite persistence) – all in a single Rust crate.

Get Started API Reference
$ cargo add nexdb-core | $ cargo nexdb serve --db mydb.nex

Everything you need, out of the box

📄

Document Store

Schemaless JSON documents with dynamic queries, indexes, and collection management.

⚡

WAL + Checkpoint

Append-only write-ahead log with snapshot-based checkpointing for crash-safety.

📡

TCP Server

Built-in serve command — connect over TCP with line-delimited JSON.

🔍

Query Builder

Fluent .where(), .sort(), .limit() API with optional JSON filter parsing.

🗄️

Indexes

B-tree indexes on any document path for high-performance lookups.

🔐

Auth & JWT

API key management and JWT-based authentication for cloud deployments.

📦

Import / Export

Bulk import & export of JSON and CSV data from the CLI and programmatically.

🧩

BSON Support

Optional binary document decoding via the bson feature flag.

🔄

Migration Engine

Dump, restore, copy, and auto-import between databases with migrate.

⚙️

Shell Completions

Generate auto-completions for bash, zsh, fish, PowerShell, and elvish.

📊

Benchmarked

Criterion-based benchmark suite for CRUD and WAL throughput.

🧹

Clean Command

Remove database files individually or wipe all databases in a directory.

See it in action

use nexdb_core::NexDb; // Open or create a database let db = NexDb::open("mydb.nex")?; // Insert a document let doc = db.collection("users").insert( serde_json::json!({"name": "Alice", "age": 30}) )?; // Query with the builder let results = db.collection("users") .query() .where_clause(r#"{"age": {"$gte": 21}}"#)? .sort("name", true) .limit(10) .run()?; // WebSocket dashboard & analytics built in let snapshot = db.analytics().snapshot()?; println!("DB has {} documents", snapshot.documents_total);
# Insert and query cargo nexdb insert mydb.nex users '{"name":"Bob","age":25}' cargo nexdb query mydb.nex users '{"age":{"$gte":21}}' # Serve over TCP cargo nexdb serve --db mydb.nex --port 9480 # Import / Export cargo nexdb import mydb.nex users data.json cargo nexdb export mydb.nex users data.json # Migrate between databases cargo nexdb migrate dump mydb.nex dump.json cargo nexdb migrate restore newdb.nex dump.json cargo nexdb migrate to-sql mydb.nex schema.sql
# Start the server cargo nexdb serve --db mydb.nex --port 9480 # Connect with netcat $ nc localhost 9480 # Every line is a JSON-RPC style request: {"method":"insert","params":{"collection":"users","document":{"name":"Charlie","age":35}}} {"ok":true,"id":"a1b2c3..."} {"method":"query","params":{"collection":"users","filter":{"age":{"$gte":21}}}} {"ok":true,"documents":[...]}
