API Reference

NexDb

The main database handle. All operations are async.

NexDb::open(path) -> Result<NexDb>

Open or create a database at the given path. Replays the WAL if the file exists.

Collection Management

MethodDescription

drop_collection(name)Drop a collection and all its documents
list_collections() -> Vec<String>List all collection names

CRUD Operations

WAL (Write-Ahead Log)

Internal persistence layer. Most users never interact with WalWriter or WalReader directly.

TypeDescription
WalEntryA single WAL record with timestamp, collection, operation, ID, and document.
WalWriterAppends entries to the WAL file with fsync.
WalReaderReads and replays WAL entries for crash recovery.

FieldIndex

An inverted index mapping field values to document IDs.

MethodDescription
new(name, field_path) -> SelfCreate a new index on a field path.
insert(doc_id, doc)Index a document.
remove(doc_id, doc)Remove a document from the index.
find_eq(value) -> Vec<String>Find document IDs with exact field value.
find_range(low, high) -> Vec<String>Find document IDs with field value in range.

Error Handling

All fallible operations return Result<T, NexDbError>.

Error VariantWhen It Occurs
NotFound(String)Document or index not found.
Duplicate(String)Insert with an existing document ID.
CollectionNotFound(String)Operation on a non-existent collection.
CollectionAlreadyExists(String)Creating a collection that already exists.
Serialization(String)JSON parse failure.
Io(std::io::Error)Filesystem error (WAL write, etc.).
Wal(String)WAL integrity or lock error.
InvalidOperation(String)Invalid state transition.
Index(String)Index creation or query error.
Json(serde_json::Error)Auto-converted serde_json error.
