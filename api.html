API Reference

NexDb

The top-level database handle. Created with NexDb::open which opens or creates a database file.

NexDb::open(path: &str) -> Result<Self>

Opens an existing database or creates a new one at the given path. The WAL file (path.wal) is also opened and any pending entries are replayed on startup.

db.collection(name: &str) -> Collection

Returns a handle to a named collection. Creates the collection if it does not exist.

db.checkpoint() -> Result<()>

Creates a full snapshot of all collections and truncates the WAL. Use this to control WAL file growth.

db.analytics() -> AnalyticsCollector

Returns an analytics collector that tracks document counts, operation counts, and cache hit rates.

db.list_collections() -> Result<Vec<String>>

Returns a list of all collection names in the database.

Collection

A named group of documents within a database. Collection handles are obtained via db.collection(name).

collection.insert(doc: Value) -> Result<String>

Inserts a JSON document and returns its auto-generated ID string.

collection.get(id: &str) -> Result<Value>

Retrieves a document by its ID. Returns an error if not found.

collection.update(id: &str, doc: Value) -> Result<()>

Replaces the document with the given ID with the new value.

collection.delete(id: &str) -> Result<()>

Deletes the document with the given ID.

collection.all() -> Result<Vec<Value>>

Returns all documents in the collection.

collection.count() -> Result<usize>

Returns the number of documents in the collection.

collection.query() -> QueryBuilder

Returns a QueryBuilder for composing filtered queries.

collection.exists(id: &str) -> Result<bool>

Checks whether a document with the given ID exists.

collection.create_index(field: &str) -> Result<()>

Creates a B-tree index on the specified document field.

collection.drop_index(field: &str) -> Result<()>

Drops the index on the specified field.

collection.list_indexes() -> Result<Vec<String>>

Lists all indexed fields in the collection.

Document

Documents are stored and returned as serde_json::Value. Each document has an _id field that is auto-generated as a UUID string if not provided.

{
  "_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "name": "Alice",
  "age": 30,
  "address": {
    "city": "Berlin",
    "zip": "10115"
  },
  "tags": ["rust", "database"]
}

When inserting, you may include your own _id value. If omitted, a UUID v4 is generated automatically.

QueryBuilder

A fluent builder for constructing and running queries. Obtained via collection.query().

builder.where_clause(json_str: &str) -> Result<Self>

Sets the filter from a JSON string. Supported operators: $eq, $ne, $gt, $gte, $lt, $lte, $in, $nin, $exists, $regex.

builder.where_value(filter: &Value) -> Result<Self>

Sets the filter from a serde_json::Value directly.

builder.sort(field: &str, ascending: bool) -> Self

Adds a sort order. Can be chained for multi-field sorting.

builder.skip(n: usize) -> Self

Skips the first n results (for pagination).

builder.limit(n: usize) -> Self

Limits results to at most n documents.

builder.run() -> Result<Vec<Value>>

Executes the query and returns the matching documents.

let results = collection
    .query()
    .where_clause(r#"{"age": {"$gte": 18, "$lt": 99}, "city": {"$in": ["Berlin", "Munich"]}}"#)?
    .sort("age", true)
    .skip(0)
    .limit(50)
    .run()?;

WAL

The write-ahead log ensures durability. Every mutation is written to the WAL before being applied to the in-memory store.

db.checkpoint() -> Result<()>

Creates a snapshot of the current database state and truncates the WAL file. Call periodically to keep WAL size bounded.

Recovery

On NexDb::open(), if the database file and a WAL file both exist, any WAL entries not yet reflected in the database are replayed automatically. This is invisible to the caller.

WAL file format

The WAL is a plain file (dbname.nex.wal) containing one JSON line per mutation. Each line has an operation type (Insert, Update, Delete) and the associated document data.

Index

NexDB uses in-memory B-tree indexes for accelerating queries on document fields.

collection.create_index(field: &str) -> Result<()>

Creates an index on the specified field path (e.g., "age" or "address.city").

collection.drop_index(field: &str) -> Result<()>

Removes the index on the given field.

collection.list_indexes() -> Result<Vec<String>>

Returns a list of all indexed field names.

Indexes are automatically updated when documents are inserted, updated, or deleted. Queries that include a filter on an indexed field will use the index for faster lookups.

MigrationEngine

The migration engine provides utilities for moving data between databases, backing up, and restoring.

MigrationEngine::new(db: &NexDb) -> Self

Creates a new migration engine bound to the given database.

engine.dump_to_file(path: &str) -> Result<()>

Dumps all collections to a portable JSON interchange file.

engine.restore_from_file(path: &str) -> Result<()>

Restores all collections from a dump file into the database.

engine.copy_to(target: &NexDb) -> Result<()>

Copies all collections from this database to another.

engine.to_sql() -> Result<String>

Generates a SQL schema string representing the database structure.

engine.auto_import(path: &str) -> Result<()>

Auto-detects the format of the given file (JSON, CSV, or dump) and imports it.

Server

The built-in TCP server accepts JSON-line requests over a TCP socket.

Server::new(db: NexDb) -> Self

Creates a new server backed by the given database.

server.with_auth(secret: &str) -> Self

Enables JWT-based authentication. Clients must include "auth": "<token>" in each request.

server.serve(host: &str, port: u16) -> Result<()>

Starts the TCP server and begins accepting connections.

Request format

{
  "method": "insert" | "get" | "update" | "delete" | "query" | "all" | "count",
  "params": { ... },
  "auth": "<jwt_token>"  // only if auth is enabled
}

Response format

{"ok": true, "id": "..."}          // insert
{"ok": true, "document": {...}}     // get
{"ok": true}                        // update / delete
{"ok": true, "documents": [...]}    // query / all
{"ok": true, "count": 42}           // count
{"ok": false, "error": "..."}       // error

Analytics

The analytics collector tracks real-time usage statistics. Useful for dashboards and monitoring.

db.analytics() -> AnalyticsCollector

Returns the collector for the database.

collector.snapshot() -> Result<AnalyticsSnapshot>

Returns a snapshot with the following fields:

FieldTypeDescription
documents_totalu64Total documents across all collections
collections_countu64Number of collections
operations_totalu64Total operations performed
cache_hitsu64Cache hit count (if caching enabled)
cache_missesu64Cache miss count

Error Handling

All fallible operations return Result<T, NexDbError>. The error enum covers:

VariantDescription
NotFoundDocument or collection not found
AlreadyExistsAttempted to create a duplicate index or document with an existing ID
InvalidIdThe provided ID is malformed or empty
InvalidFilterThe query filter JSON is invalid or contains unsupported operators
StorageAn underlying I/O or storage error occurred
SerdeJSON serialization or deserialization error
AuthAuthentication failure (invalid or missing JWT token)
ConfigConfiguration or runtime environment error
InternalAn internal invariant was violated (should not happen under normal use)
