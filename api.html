API Reference

Complete reference for the NexDB public API, from the top-level NexDb handle to the query builder, TCP server, and migration engine.

NexDb

The top-level database handle. All operations begin here.

NexDb::open(path: &str) -> Result<Self>

Opens an existing database or creates a new one at the given path. The accompanying WAL file (path.wal) is replayed automatically on startup for crash recovery.

db.collection(name: &str) -> Collection

Returns a handle to a named collection. Creates the collection if it does not already exist.

db.checkpoint() -> Result<()>

Creates a full snapshot of all in-memory collections and truncates the WAL file. Call periodically to keep WAL growth bounded.

db.analytics() -> AnalyticsCollector

Returns an analytics collector that tracks document counts, operation counts, and cache metrics.

db.list_collections() -> Result<Vec<String>>

Returns a list of all collection names in the database.

Collection

A named group of documents. Obtained via db.collection(name).

collection.insert(doc: Value) -> Result<String>

Inserts a JSON document. Returns the auto-generated _id (UUID v4). Provide your own _id in the document to override.

collection.get(id: &str) -> Result<Value>

Retrieves a document by ID. Returns Err(NotFound) if the document does not exist.

collection.update(id: &str, doc: Value) -> Result<()>

Replaces the document at id with the new value.

collection.delete(id: &str) -> Result<()>

Deletes the document at id.

collection.all() -> Result<Vec<Value>>

Returns every document in the collection.

collection.count() -> Result<usize>

Returns the number of documents in the collection.

collection.query() -> QueryBuilder

Returns a QueryBuilder for filtered, sorted, and paginated queries.

collection.exists(id: &str) -> Result<bool>

Checks whether a document with the given ID exists.

collection.create_index(field: &str) -> Result<()>

Creates a B-tree index on the specified field path, for example "age" or "address.city".

collection.drop_index(field: &str) -> Result<()>

Drops the index on the given field.

collection.list_indexes() -> Result<Vec<String>>

Lists all indexed field names for the collection.

Document

Documents are stored as serde_json::Value. Every document includes an _id field, auto-generated as a UUID v4 when not provided on insert.

{
  "_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "name": "Alice",
  "age": 30,
  "address": { "city": "Berlin", "zip": "10115" },
  "tags": ["rust", "database"]
}

Documents can contain nested objects, arrays, strings, numbers, booleans, and null values — anything serde_json::Value supports.

QueryBuilder

Fluent builder for composing queries. Obtained via collection.query().

builder.where_clause(json_str: &str) -> Result<Self>

Sets the filter from a JSON string. Supports operators: $eq, $ne, $gt, $gte, $lt, $lte, $in, $nin, $exists, $regex.

builder.where_value(filter: &Value) -> Result<Self>

Sets the filter from a pre-parsed serde_json::Value.

builder.sort(field: &str, ascending: bool) -> Self

Adds a sort order. May be chained for multi-field sorting.

builder.skip(n: usize) -> Self

Skips the first n results (for pagination).

builder.limit(n: usize) -> Self

Limits results to at most n documents.

builder.run() -> Result<Vec<Value>>

Executes the query and returns matching documents.

let results = collection
    .query()
    .where_clause(r#"{"age": {"$gte": 18, "$lt": 99}}"#)?
    .sort("age", true)
    .skip(0)
    .limit(50)
    .run()?;

WAL

The write-ahead log ensures durability. Every mutation is appended to the WAL before it is applied in memory.

db.checkpoint() -> Result<()>

Creates a snapshot of the full in-memory state and truncates the WAL. Bounds WAL size without sacrificing safety.

Automatic recovery

On NexDb::open(), any WAL entries not yet reflected in the database are replayed automatically. This process is invisible to the caller.

WAL file format

The WAL is stored at dbname.nex.wal. Each line is a JSON object encoding one mutation:

{"type":"Insert","id":"uuid","document":{...}}
{"type":"Update","id":"uuid","document":{...}}
{"type":"Delete","id":"uuid"}

Index

In-memory B-tree indexes for accelerating queries on document fields. Indexes are maintained automatically — when a document is inserted, updated, or deleted, every index for that collection is updated transparently.

When a query filter includes an indexed field, the engine narrows results via the index (O(log n)) before applying any remaining filter conditions.

Functions: create_index(field), drop_index(field), list_indexes() — all available on Collection.

MigrationEngine

Utilities for moving data between databases, creating backups, and generating SQL schemas.

MigrationEngine::new(db: &NexDb) -> Self

engine.dump_to_file(path: &str) -> Result<()>

Dumps all collections to a portable JSON interchange file with metadata.

engine.restore_from_file(path: &str) -> Result<()>

Restores all collections from a dump file into the database.

engine.copy_to(target: &NexDb) -> Result<()>

Copies all collections from the current database to another.

engine.to_sql() -> Result<String>

Generates a SQL schema string with CREATE TABLE and INSERT statements.

engine.auto_import(path: &str) -> Result<()>

Auto-detects the input format (JSON array, CSV with headers, or a dump file) and imports accordingly.

Server

The built-in TCP server accepts JSON-line requests over a TCP socket. It can be configured programmatically or started from the CLI via serve.

Server::new(db: NexDb) -> Self

server.with_auth(secret: &str) -> Self

Enables JWT authentication. Clients must include "auth": "<token>" in each request.

server.serve(host: &str, port: u16) -> Result<()>

Starts the TCP server on the given host and port. Blocks the current thread.

Request and response format

// request
{
  "method": "insert" | "get" | "update" | "delete" | "query" | "all" | "count",
  "params": { ... },
  "auth": "<jwt_token>"
}

// responses (one JSON line per request)
{"ok": true, "id": "..."}          // insert
{"ok": true, "document": {...}}     // get
{"ok": true}                        // update / delete
{"ok": true, "documents": [...]}    // query / all
{"ok": true, "count": 42}           // count
{"ok": false, "error": "..."}       // error

Analytics

Real-time usage statistics suitable for dashboards and monitoring.

db.analytics() -> AnalyticsCollector

collector.snapshot() -> Result<AnalyticsSnapshot>

Returns a snapshot containing:

FieldTypeDescription
documents_totalu64Total documents across all collections
collections_countu64Number of collections
operations_totalu64Total operations performed
cache_hitsu64Cache hit count
cache_missesu64Cache miss count

Error Handling

All fallible operations return Result<T, NexDbError>.

VariantDescription
NotFoundDocument or collection not found
AlreadyExistsDuplicate index or document ID
InvalidIdMalformed or empty ID
InvalidFilterInvalid query filter or unsupported operator
StorageUnderlying I/O or storage error
SerdeJSON serialization or deserialization error
AuthAuthentication failure (invalid JWT)
ConfigConfiguration or runtime environment error
InternalInternal invariant violation
