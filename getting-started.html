Getting Started

Everything you need to install NexDB, write your first application, and use every feature — from the Rust API to the CLI to the TCP server.

Installation

Add the core library to your Cargo.toml:

[dependencies]
nexdb-core = "0.1.0"
serde_json = "1.0"

Enable optional BSON support:

nexdb-core = { version = "0.1.0", features = ["bson"] }

Install the CLI tool:

cargo install nexdb-cli

Or run directly with cargo run:

cargo nexdb --help

Quick Start

Open a database, insert a document, and query it — in under 10 lines of Rust:

use nexdb_core::NexDb;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let db = NexDb::open("mydb.nex")?;

    let collection = db.collection("users");
    let id = collection.insert(serde_json::json!({
        "name": "Alice",
        "age": 30,
        "city": "Berlin"
    }))?;

    let user = collection.get(&id)?;
    println!("Found: {}", user);
    Ok(())
}

Same thing via the CLI:

cargo nexdb insert mydb.nex users '{"name":"Alice","age":30,"city":"Berlin"}'
cargo nexdb get mydb.nex users <id>

CRUD Operations

Insert

let id = collection.insert(json!({"title": "Hello", "views": 42}))?;
// Returns the auto-generated _id (UUID v4)

Get by ID

let doc = collection.get(&id)?;
// Returns serde_json::Value

Update

collection.update(&id, json!({"views": 43}))?;
// Replaces the entire document

Delete

collection.delete(&id)?;

List all

for doc in collection.all()? {
    println!("{:?}", doc);
}

Count

let count = collection.count()?;
println!("{} documents", count);

Exists

if collection.exists(&id)? {
    println!("Document exists");
}

Query Builder

The QueryBuilder provides a fluent API for composing rich queries with filters, sorting, and pagination — inspired by MongoDB's query language.

use nexdb_core::query::QueryBuilder;

let results: Vec<serde_json::Value> = collection
    .query()
    .where_clause(r#"{"age": {"$gte": 21, "$lt": 65}, "city": "Berlin"}"#)?
    .sort("name", true)
    .skip(0)
    .limit(20)
    .run()?;

You can also pass a parsed serde_json::Value directly:

let filter: serde_json::Value =
    serde_json::from_str(r#"{"age": {"$gte": 21}}"#)?;
let results = collection
    .query()
    .where_value(&filter)?
    .run()?;

Indexes

B-tree indexes accelerate queries on document fields. Created and managed per-collection.

// Create an index on the "age" field
collection.create_index("age")?;
collection.create_index("address.city")?;

// Query automatically uses the index
let results = collection.query()
    .where_clause(r#"{"age": {"$gte": 21}}"#)?
    .run()?;

// List and drop
let indexes = collection.list_indexes()?;
collection.drop_index("age")?;

Without an index: full collection scan (O(n)). With an index on the filtered field: O(log n) via the B-tree. Indexes are maintained automatically on insert, update, and delete.

WAL & Checkpoint

NexDB uses a write-ahead log to guarantee crash safety. Every mutation is written to the WAL before it is applied in memory.

// WAL is enabled by default — automatic and transparent
// Manually checkpoint (snapshot + truncate WAL)
db.checkpoint()?;

Automatic recovery: On NexDb::open(), if a WAL file (mydb.nex.wal) exists alongside the database file, all pending entries are replayed before the database is made available.

Checkpointing serializes the full in-memory state as a snapshot entry and truncates the WAL. Call periodically to keep the WAL file size bounded. Safe to call at any time.

TCP Server

Start a TCP server that accepts JSON-line requests:

cargo nexdb serve --db mydb.nex --port 9480 --host 127.0.0.1

Each line is a JSON object with method and params:

{"method":"insert","params":{"collection":"users","document":{"name":"Charlie","age":35}}}
{"method":"get","params":{"collection":"users","id":"a1b2c3..."}}
{"method":"query","params":{"collection":"users","filter":{"age":{"$gte":21}}}}
{"method":"delete","params":{"collection":"users","id":"a1b2c3..."}}
{"method":"count","params":{"collection":"users"}}
{"method":"all","params":{"collection":"users"}}

Responses are JSON-line objects:

{"ok":true,"id":"a1b2c3..."}
{"ok":true,"document":{...}}
{"ok":true,"documents":[...]}
{"ok":true,"count":42}
{"ok":false,"error":"Document not found"}

Enable JWT authentication:

cargo nexdb serve --db mydb.nex --require-auth --jwt-secret "mysecret"

Clients include an "auth" field with their JWT token in each request.

Import / Export

Bulk operations for JSON and CSV formats. The CLI commands are straightforward:

# JSON
cargo nexdb import mydb.nex users data.json
cargo nexdb export mydb.nex users data.json

# CSV (first row = headers)
cargo nexdb import-csv mydb.nex users data.csv
cargo nexdb export-csv mydb.nex users data.csv

Programmatic equivalents:

let docs = db.collection("users").all()?;
let json = serde_json::to_value(&docs)?;
std::fs::write("users.json",
    serde_json::to_string_pretty(&json)?)?;

Migration Engine

The migrate command family handles moving data between databases and generating SQL schemas.

# Dump to portable JSON interchange format
cargo nexdb migrate dump mydb.nex backup.json

# Restore into a new or existing database
cargo nexdb migrate restore newdb.nex backup.json

# Copy directly between two databases
cargo nexdb migrate copy source.nex target.nex

# Generate SQL schema (CREATE TABLE + INSERT)
cargo nexdb migrate to-sql mydb.nex schema.sql

# Auto-import — detect format and import
cargo nexdb migrate auto-import mydb.nex data.json

Programmatic usage:

use nexdb_core::migrate::MigrationEngine;

let engine = MigrationEngine::new(&db);
engine.dump_to_file("backup.json")?;
engine.restore_from_file("backup.json")?;
let sql = engine.to_sql()?;
println!("{}", sql);

Shell Completions

Generate auto-completion scripts for your shell so you never have to type a full command again.

# Bash
cargo nexdb completions bash > /etc/bash_completion.d/nexdb

# Zsh
cargo nexdb completions zsh > /usr/local/share/zsh/site-functions/_nexdb

# Fish
cargo nexdb completions fish > ~/.config/fish/completions/nexdb.fish

# PowerShell
cargo nexdb completions powershell > profile.ps1

# Elvish
cargo nexdb completions elvish > ~/.elvish/completions.eld

Clean Command

Remove database files cleanly when they are no longer needed.

# Remove a specific database + its WAL
cargo nexdb clean mydb.nex

# Remove ALL NexDB databases in a directory
cargo nexdb clean --all ./data/

BSON Support

When the bson feature is enabled, NexDB can decode BSON binary documents into its internal JSON representation.

[dependencies]
nexdb-core = { version = "0.1.0", features = ["bson"] }

BSON documents inserted via the TCP server or programmatic API are automatically decoded. The feature is off by default, keeping the dependency footprint minimal.

Benchmarks

NexDB includes a criterion benchmark suite. Run it to see real-world performance on your hardware.

# Run all benchmarks
cargo bench

# Run a specific benchmark group
cargo bench --bench crud
cargo bench --bench wal

Benchmarks cover: insert throughput, read latency, update performance, delete performance, indexed queries, and WAL write throughput. Results are reported in operations/second with latency percentiles.
